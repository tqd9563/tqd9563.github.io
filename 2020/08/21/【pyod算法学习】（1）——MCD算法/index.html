1. MSD异常检测

        关于单总体的异常检测场景，一个著名的方法是计算每个样本点到全体数据“中心”的距离。如果样本到中心的距离大于某个设定好的阈值，则这个样本可能是一个异常点。在多元数据中，一种定义样本点到样本集中心的二次平方距离如下：

这里$X$是某个样本，$Y$是样本全体的中心（Location），$S$是样本集协方差矩阵（Shape），这个距离被叫做Mahalanobis平方距离（MSD）。MSD越大，样本越可能异常。

        在聚类的异常检测场景中，通过对原始数据应用某个稳健的聚类算法后，得到的各个聚类簇就可以被看作是来自一个个不同的独立的总体，而异常检测算法就可以在每个簇上单独使用，因此一个异常点通常被认为是那些到每一个聚类簇的中心的MSD都很大的点。

        然而，对于来自单总体的样本，传统的样本均值和协方差估计方法对异常值会很敏感，Hardin和Rocke(2002)提出了一种利用均值和协方差的稳健估计来对MSD进行的分布拟合，这种方法称作MCD（minimum covariance determinant， 最小协方差矩阵行列式）。

2. MCD的定义与推广

2.1 单总体MCD的定义

        对于单总体场景，给定$n$个样本，这$n$个样本的MCD指的是其中某$h$个样本的样本均值和协方差矩阵，这$h$个样本需要满足其协方差矩阵行列式是所有样本量为$h$的可能样本组合中最小的。数学语言描述如下：

        这里的参数$h$可以理解为全部$n$个样本中一定有至少$h$个不是偏僻样本。当$h=\lfloor(n+p+1)/2\rfloor$时，MCD有最高的可能breakdown，后面的讨论中$h$都是用这个值，并把所有样本量为$h$的样本集合叫做half sample。因为MCD是根据最“紧凑”的一组half sample计算得到的，所以异常点不会对其造成偏移影响。

2.2 推广到多重聚类

        上面的MCD概念是基于单总体的，这很容易推广到多重聚类的场景。假设经过一个不错的聚类初始化，并且知道了聚类簇的个数$g$，那么MCD就可以分别应用在这$g$个簇上，每个簇的大小由那些离这个簇最近的样本个数决定，所以每个簇的大小和每个簇对应的MCD样本数分别为$n_i, h_i=\lfloor(n_i+p+1)/2\rfloor$。

        单总体情形和多重聚类情形的区别在于，后者的MCD样本需要对每一个簇分别计算一次，因此一个良好的稳健的聚类初始化算法是很重要的。

2.3 MCD的优点

        MCD的均值和协方差估计常被用作聚类簇的中心和形状估计。此时，对于某个具体的聚类簇被判定为异常点的样本，将不会参与估计这个簇的中心和形状（因为这个簇的中心和形状估计用的是到这个簇的MSD距离最小的$h$个样本，而异常点的MSD距离一定非常大）。进一步，对于所有的聚类簇都被判定是异常点的样本，则将不会被用来估计任何一个簇的中心和形状。

3. MCD估计

        实际情况中除非是小样本，否则样本集精确的MCD是无法求得的，因此用来估计MCD的算法就是我们的MCD估计。对于多重聚类场景和单总体场景，MCD的估计算法是相似的，区别只是在于前者的算法初始点不是随机选择的样本子集（理由见paper）。异常检测方法并不依赖于特定的稳健聚类初始化算法，换句话说，任何一种稳健的初始化算法最终都会得到相近的异常检测结果。

3.2 多重聚类场景下对各个聚类簇计算MCD估计

  1. 首先确定一共有几个聚类簇$g$
  2. 使用某种稳健的聚类算法进行初始化，得到$g$个聚类簇，然后计算每个簇的均值和协方差阵（单总体step2）
  3. 计算每个样本点到各个簇的MSD，并根据MSD将每个样本重新分配到各个簇（每个样本被分配到其最小的MSD对应的那个簇），最后得到每个簇的大小$n_j$（单总体step3）
  4. 对重新得到的每个簇$j$，选出各自最小的$h_j$个MSD所对应的half sample（单总体step4 - 5）
  5. 对每个簇，利用刚才得到的half sample重新计算一次均值和协方差阵
  6. 重复步骤3 - 5，直到每个簇的half sample不再改变。

4. MCD应用于异常检测

        在MSD的计算中使用MCD估计后，可以得到每个样本点到每个聚类簇中心的稳健距离。在第一节里提到过，一个异常点理应到每个聚类中心的距离都比较大。然而事实上并不是所有的数据集都能够在属于数据集的极端点（非异常点）和不属于数据集的极端点（异常点）之间产生明显的分界线。而为了区分这两种极端点，异常点可以用一个关于聚类的$F$分布的分位数来确定，而一个样本点被认为是异常点，当且仅当它关于每一个聚类簇都是异常的。

5. 利用pyod实现MCD算法

        pyod包里的所有模型的API使用手感都和sklearn相近，上手很快。MCD算法在pyod中的实现是pyod.model.mcd.MCD类：

几个参数的解释如下：

  • contamination：所有模型都有的参数， 也是无监督学习绕不开的参数 ，意思就是数据集中异常点的比例，这个是用来乘以样本量然后给模型输出的得分来划分阈值用的。
  • store_precision：没什么特别的含义
  • assume_centered：对那些均值显著接近等于0又不确切等于0的数据可以设置为True，这样就不会中心化数据直接计算协方差估计。默认为False，此时均值和协方差的稳健估计会直接用FastMCD算法求得。
  • support_fraction：论文中的参数$h$，默认等于$(\lfloor n_sample + n_feature - 1\rfloor / 2)$。 实际调参时这个参数需要注意，可能因为数据原因这个值如果取的小了模型会报错 = =
  • random_state：没啥好说的，随机种子。。而且不同随机种子出来的模型结果差距还是挺大的，可以考虑做复数次训练。

        pyod里的模型类的方法基本都是一样的，常用的就下面几个：

  • fit(X, y=None)：训练模型，只要输入训练数据$X$即可，没有标签
  • predict(X)：预测一个样本是否是异常点
  • predict_proba(X, method="linear")：预测一个样本是否是异常点，输出的是一个概率矩阵。其实pyod的算法本身输出的是outlier score，而predict_proba()方法本质上是将score映射到$[0, 1]$区间。默认的method = "linear"，文档里写的是用简单的min - max方法将score归一化，不过我自己手动min - max出来的结果和他的还是有出入的…不清楚原因；另外可选的是method = "unify"

        常用的属性也就下面几个：

  • threshold：基于参数contamination的阈值，是样本的desicion_score_里排名最高的前n\_samples * contamination个样本

  • decision_scores_：训练数据的outlier score，array形式

  • labels_：训练数据的二元标签，0表示正常样本，1表示异常样本。计算方法就是将阈值应用在decision_score_上。
